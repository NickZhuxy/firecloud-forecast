## 1. 问题:为什么现在会错

`canvas_layer_from_diagnosis`(illumination.py:88)= `max(layers, key=base_m)` ——
诊断出的层里**永远取最高的当画布**。讽刺的是,旧的非诊断路径 `select_canvas_layer`
(features.py:72)反而是按分层云量选的;#13 引入真实诊断层后,选层这一步退化成了
纯高度准则。后果(审计验收反例):一层 5% 覆盖、τ≈0 的卷云丝,会把下方 80% 成片
中云挤下画布位——随后云底、时长、高度偏好、遮挡、覆盖打分全部键到那层丝上。
现有遮挡机制只能部分兜底(把"高画布被下层挡死"折进分数),但对天空的**描述**
是错的:真正会烧的是中云那层。

## 2. 手册的物理(照录 + 最小重述)

### 2.1 伊春算例的选层原文(§4.1.1)

> "当然也可以查看高云云量预报,但是这个案例里面在看过了以后发现**高云云量没有
> 中云多,而且高云边界比中云边界近,所以直接看中云云量就行了**。"

两条准则、同向裁决:**云量**(该层覆盖率大者代表实际的天空表现)与**边界距离**
(沿日落方向边界更远者=云盖延展更足,是"这场秀的主体")。注:手册这里比较的是
Windy 的中/高云量**图**——同一物理云盖在不同分层图上的注册强弱;推广到我们的
管线,即对**诊断出的每一层**,取它所属 étage 的同源云量(GFS LCDC/MCDC/HCDC,
`for_tier(tier_from_height(base))`,#35 的同源原则)与沿日落方向的该 étage 边界。

### 2.2 深圳算例:低云可以当画布,但薄=不稳健(§4.1.1)

- 深圳案例整场的画布就是 2036 m 的**层积云(低云)**——前提是当时**没有**中高云。
  ⇒ 低云画布合法,但仅当没有"在场"的中高云盖(中高云在场时低云是遮挡,#13 已定)。
- "层积云的厚度薄,而且云内相对湿度最大值也没有饱和……**云量的不确定性被放大了**"
  ⇒ 厚度/光学厚度进选层的角色是**稳健性**(薄→到时候可能没云),不是显色好坏
  (薄卷云恰恰显色最好)。§4.1.2(b) 同理:薄中云→湿度不匀→云洞。
- "如果云在其他方向上边界过于靠近本地会怎样,**云量会不会太少**"⇒ 边界过近 =
  本地云量边缘化,与 2.1 的边界准则同源。

### 2.3 高度偏好还在,但降为软准则

层越高被照亮越久(§1.2 几何;theory 笔记 cloud-physics `CloudAltitudePreference`:
高 1.0 / 中 0.5 / 低 0.1)。这是**同等表现下**偏好更高层的理由,不是让 5% 丝云
压过成片中云的理由——后者正是本条要修的 bug。

## 3. 设计:两步选层 + 乘法多准则分

**第 1 步(étage 优先权,保 #13 语义)**:候选 = 覆盖率未知**或** ≥ 在场阈值
(10%,镜像 `features._LAYER_PRESENCE_THRESHOLD`,因 features→illumination 的
import 方向不能反向引用而本地定义)的**中/高**层;若无此类候选,退化为全部层
(低云此时才有资格——深圳案例)。

**第 2 步(多准则分)**:`score = cover · substance · height · extent`,取 argmax,
平分时取更高 base(与旧行为一致的确定性平局规则):

| 项 | 定义 | 角色 |
|---|---|---|
| cover | `cover_pct/100`(未知→1.0 中性) | **主导**:云量,线性惩罚稀薄覆盖 |
| substance | `0.5 + 0.5·opacity`(复用 `_layer_opacity`:τ 优先、厚度×相态兜底、置信加权) | 稳健性:压真丝云,地板 0.5 保"薄而真"的卷云仍可胜 |
| height | `0.6 + 0.4·min(base_m/8000, 1)` | 软偏好:连续无台阶,越高越受照 |
| extent | `0.5 + 0.5·(boundary_km/max(boundary among candidates))`(未知→1.0 中性;该层在剖面内未见边界→按剖面最远处理,"云盖延展出数据范围") | 边界距离:更远=延展更足(伊春准则) |

- 选层用**原始**(未平流)边界:选层是稳健性比较;FA-T1 的平流订正仍只作用于
  选定画布的门控路径。
- cover 只取 **GFS 同源** `CloudCover`(#35:用 OM 云量给 GFS 诊断层排序会重蹈
  跨源零化);无 GFS cover 时 cover 项整体退出(全 1.0)。
- cover=0 的诊断层(廓线见云、云量场说无)被第 1 步排除——同源云量场是仲裁者;
  这是 #35 原则的自然延伸,记录之。
- 无任何辅助数据(bare `layers`)时:中/高层全候选,substance×height 排序——
  与旧"取最高"在单层、同质多层情形完全一致;丝云 vs 实盖情形**故意**改判
  (这正是审计要的行为变化)。

### 数量级自检

- 验收反例:5% τ≈0 卷云(conf 0.5)vs 80% τ=3 中云(conf 0.8,base 4 km):
  `0.05·0.5·1.0 ≈ 0.025` vs `0.8·0.88·0.8 ≈ 0.56` → 中云,~22×。✓
- 整层薄卷云真画布:HCDC 100% τ=0.1 vs MCDC 60% τ=3:
  `1.0·0.54·1.0 ≈ 0.54` vs `0.6·0.88·0.8 ≈ 0.42` → 卷云。薄≠出局。✓
- 伊春构型(HCDC<MCDC 且高云边界更近):cover 与 extent 同向 → 中云。✓

## 4. 验证设计(先写失败测试)

Metamorphic/性质不变量(全离线合成层):

1. **覆盖率方向律**:抬高某层 étage 覆盖率(其余不动)永不把该层从画布上拉下来。
2. **验收反例**:5% 丝云 + 80% 中云 → 画布=中云;覆盖率对调 → 画布=丝云。
3. **étage 优先权**:100% 低云 + ≥10% 中/高 → 画布=中/高(低云是遮挡,#13);
   中/高覆盖 <10% 时 → 低云可当画布(深圳案例);只有低云 → 低云(零回归)。
4. **高度偏好退化**:覆盖率/τ/厚度全同 → 更高层胜(旧行为的退化保持);
   单层天空恒为画布,与数据多寡无关。
5. **边界方向律**:把某层边界移远(其余不动)永不把该层从画布上拉下来;
   伊春构型(cover+边界同向)选中云。
6. **NaN/None 安全**:τ=NaN、cover=None、boundary=None 任意组合不产生 NaN 序,
   结果确定。
7. **不翻转⇒零扰动**:选层结果与旧准则一致的天空,下游 Forecast 数值(概率、
   几何、遮挡)逐位不变——接线不得扰动非翻转情形。
8. **平局确定性**:完全平分 → 更高 base,稳定可复现。

对照:§2.1 伊春构型、§2.2 深圳低云画布;`test_metamorphic_physics` 按 #57 验收扩充。
